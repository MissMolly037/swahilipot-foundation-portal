## Versioning

This project follows **Semantic Versioning**:

* **MAJOR** version for incompatible changes.
* **MINOR** version for new features that remain backward compatible.
* **PATCH** version for backward-compatible bug fixes.

Example:

* `1.0.0`
* `1.1.0`
* `1.1.1`
