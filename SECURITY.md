# Security Policy

Thank you for helping keep the **Swahilipot Foundation Portal** secure.

We take security seriously and appreciate responsible disclosure of potential vulnerabilities.

---

# Supported Versions

The following table indicates which versions of the project currently receive security updates.

| Version              | Supported |
| -------------------- | --------- |
| Latest (main branch) | ✅ Yes     |
| Older releases       | ❌ No      |

---

# Reporting a Security Vulnerability

If you discover a security vulnerability, **please do not open a public GitHub issue**.

Instead, report it privately by contacting the project maintainer with:

* A description of the vulnerability
* Steps to reproduce the issue
* The potential impact
* Any suggested fix (if available)

Please include screenshots, logs, or proof-of-concept examples where appropriate.

---

# What to Expect

After receiving a security report, we aim to:

* Acknowledge receipt within a reasonable timeframe.
* Investigate the reported issue.
* Validate the vulnerability.
* Develop and test a fix.
* Release a security update if necessary.
* Credit the reporter where appropriate (unless anonymity is requested).

---

# Security Best Practices

When contributing to this project, please:

* Never commit passwords, API keys, or secrets.
* Store sensitive values in environment variables (`.env`).
* Keep project dependencies up to date.
* Follow Django security best practices.
* Validate all user input.
* Apply authentication and authorization checks where required.

---

# Third-Party Dependencies

This project relies on trusted open-source packages.

Please report any known vulnerabilities affecting project dependencies so they can be reviewed and updated promptly.

---

# Scope

This policy applies to:

* Source code
* Documentation
* Configuration files
* Deployment configurations
* Third-party integrations
