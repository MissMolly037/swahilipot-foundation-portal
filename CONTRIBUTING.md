# Contributing to Swahilipot Foundation Portal

Thank you for your interest in contributing to this project.

## Getting Started

1. Fork the repository.
2. Clone your fork.
3. Create a new feature branch.
4. Install the project dependencies.
5. Make your changes.
6. Test your changes before committing.

## Branch Naming

Use descriptive branch names.

Examples:

- feature/user-management
- feature/event-calendar
- bugfix/login-error
- docs/update-readme

## Commit Messages

Follow clear commit messages.

Examples:

- feat: add attendance reports
- fix: resolve login validation
- docs: update installation guide
- refactor: simplify dashboard logic

## Pull Requests

Before opening a pull request:

- Ensure the application runs successfully.
- Keep changes focused on a single feature or fix.
- Update documentation if needed.

## Code Style

- Follow Django best practices.
- Write readable, maintainable code.
- Use meaningful variable and function names.

## Reporting Issues

When reporting an issue, include:

- Description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

Thank you for helping improve the Swahilipot Foundation Portal.
