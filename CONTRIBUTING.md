# Contributing to Swahilipot Foundation Portal

First of all, thank you for considering contributing to the **Swahilipot Foundation Portal**. Every contribution, whether it's a bug fix, new feature, documentation improvement, or performance enhancement, is greatly appreciated.

## Code of Conduct

By participating in this project, you agree to contribute in a respectful, collaborative, and professional manner. Please help us maintain a welcoming environment for everyone.

## Getting Started

1. Fork the repository.
2. Clone your fork to your local machine.
3. Create a new feature branch.
4. Install the project dependencies.
5. Configure your local environment.
6. Run the project locally.
7. Make your changes.
8. Test your changes thoroughly before committing.

## Branch Naming Convention

Use clear and descriptive branch names.

Examples:

* feature/user-management
* feature/event-calendar
* feature/report-generation
* bugfix/login-error
* bugfix/dashboard-crash
* docs/update-readme
* refactor/improve-authentication

## Commit Message Convention

Use meaningful commit messages following conventional commit standards.

Examples:

* feat: add attendance reports
* feat: implement QR code check-in
* fix: resolve login validation
* fix: correct event registration bug
* docs: update installation guide
* style: improve dashboard layout
* refactor: simplify dashboard logic
* test: add unit tests for authentication

## Pull Requests

Before opening a Pull Request, please ensure that:

* The application runs successfully.
* All tests pass.
* Your code follows the project's coding standards.
* Documentation has been updated where necessary.
* Your Pull Request focuses on a single feature or bug fix.

## Code Style Guidelines

Please follow these best practices:

* Follow Django and Python best practices.
* Write clean, readable, and maintainable code.
* Use meaningful variable, class, and function names.
* Keep functions and methods focused on a single responsibility.
* Comment only when necessary to explain complex logic.

## Reporting Issues

When creating an issue, please include:

* A clear title
* A detailed description
* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots (if applicable)
* Operating system and browser information (if relevant)

## Feature Requests

Feature suggestions are always welcome. When proposing a new feature, please describe:

* The problem it solves
* Your proposed solution
* Any alternative approaches you considered

## Questions

If you have questions about contributing, feel free to open a discussion or create an issue in the repository.

---

Thank you for helping improve the **Swahilipot Foundation Portal**. Your contributions help make this project better for everyone.
